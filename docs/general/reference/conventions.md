# Conventions

Repo-wide conventions shared by every area. The terse rule form lives in the root [CLAUDE.md](../../../CLAUDE.md); this page carries the detail and the *why*.

## Commits

- One imperative line: `add X`, `fix Y`, `update Z`. No body unless it earns its keep.
- NEVER append `Co-Authored-By` or "Generated with Claude" (or any tool) attribution.
- NEVER force-push.

Why: history stays clean, linear, and authored by the human — bisect, blame, and review all assume that. Attribution trailers and rewritten history break that assumption.

## Pull requests

- PRs target `main` and are **draft by default**; mark ready when it is.
- One exception: the Flux image-automation flow. Flux's image-automation controller pushes `image-automation-<cluster>-<app>` branches, and CI opens the PR for them automatically (parsed into a titled, reviewer-assigned PR). These are the only PRs you do not open by hand. See [ci.md](ci.md).

## YAML: document markers + prettier

Every **hand-authored** YAML file under `homelab/` must:

1. Begin with a `---` document start marker (first non-comment line).
2. Be prettier-formatted per [`.prettierrc`](../../../.prettierrc): 2-space indent, `singleQuote: true`, `trailingComma: none`, `bracketSpacing: true`, `proseWrap: preserve`.

### Exemptions

These directories are skipped by both the marker script and the formatter:

| Path | Why exempt |
|------|-----------|
| `flux-system/` | Flux-generated; owned by `flux bootstrap`, not hand-edited. |
| `talos/` | Machine configs / secrets generated by `talosctl`. |
| `terraform/` | Provider-generated / tool-owned YAML. |

Only `homelab/clusters/tom/flux-system/` is also listed in [`.prettierignore`](../../../.prettierignore); the CI file filters exclude all three by path.

### Why CI, not local hooks

There is no local pre-commit hook. The [format workflow](ci.md) enforces both rules on every PR: [`scripts/add-yaml-markers.sh`](../../../scripts/add-yaml-markers.sh) inserts missing `---` markers (skipping `flux-system/`, `talos/`, `terraform/`), then prettier reformats. Both jobs commit their fixes back to the PR branch as `github-actions[bot]`. Keeping the rule in CI means it applies uniformly regardless of who (or what) authored the change, and contributors need nothing installed.

## Secrets

- Never commit real secrets. The gitignored Talos files under `homelab/infrastructure/talos/` are enforced by [`homelab/infrastructure/talos/.gitignore`](../../../homelab/infrastructure/talos/.gitignore): `secrets.yaml`, `talosconfig`, `controlplane*.yaml`, and `worker*.yaml` (the `*.patch.yaml` inputs are intentionally kept).
- Root [`.gitignore`](../../../.gitignore) also excludes `**/id_rsa` and `terminal/bench-results/`.
- Cluster secrets are never stored in git. They live in 1Password (the `Homelab - Tom` vault) and are pulled into the cluster by the External Secrets Operator, which authenticates with a 1Password service-account token seeded once as the `1password-service-token-tom` Kubernetes secret.
